// 1-Month Learning Roadmap

Habit Tracker Project Roadmap

Built and authored by Prasann · Prasannova Solutions
ASP.NET MVC → Identity → Google Login → EF Core → Google Fit → Dashboard

ASP.NET MVC Entity Framework Core SQL Server ASP.NET Identity Google OAuth 2.0 Google Fit API Bootstrap 5 Chart.js
WEEK 01 Foundation — MVC, EF Core & Database Days 1–7
📘 Learn First
  • MVC project structure (Controllers, Views, Models)
  • Razor syntax — @Model, @foreach, forms
  • EF Core: DbContext, Migrations, relationships
  • Code-First approach: one-to-many (User → Habits)
  • LINQ basics: .Where(), .ToList(), .FirstOrDefault()
🔨 Then Build
  • Create Habit, HabitLog models
  • Run first migration → SQL Server DB created
  • CRUD for Habits (List, Create, Edit, Delete)
  • Mark habit as done today (checkbox → POST)
  • View weekly habit table like the image
✅

Week 1 Goal: You can add habits and mark them done each day. The grid table (like your screenshot) shows checkmarks per date.

WEEK 02 Auth — Identity + Google Login Days 8–14
📘 Learn First
  • ASP.NET Identity — scaffold, UserManager, SignInManager
  • What is OAuth 2.0? (just the concept, 30 min)
  • Google Cloud Console → create OAuth credentials
  • Configure AddGoogle() in Program.cs
  • Protect routes with [Authorize] attribute
  • ClaimsPrincipal — get logged-in user ID
🔨 Then Build
  • Scaffold Identity pages (Login, Register)
  • Add "Login with Google" button
  • Tie Habits to logged-in user (UserId FK)
  • Each user sees only their own habits
  • Nav bar: show username + logout
✅

Week 2 Goal: Users can register, log in with Google, and each sees their own private habit list. No more shared data.

WEEK 03 Google Fit Integration — Calories & Distance Days 15–21
📘 Learn First
  • What is Google Fit REST API? (read docs overview)
  • OAuth scopes for Fitness — fitness.activity.read
  • How to store OAuth tokens in DB (refresh token)
  • HttpClient in C# — GET request to Fit API
  • Parse JSON response with System.Text.Json
  • Aggregate data points — calories, steps, distance
🔨 Then Build
  • Request fitness scope on Google login
  • Save access + refresh tokens to DB
  • GoogleFitService — fetch today's calories & km
  • Show calories burned on dashboard card
  • Show km distance on dashboard card
  • Set daily goals (user can edit target kcal / km)
✅

Week 3 Goal: Dashboard shows today's calories burned and km walked from Google Fit. User can set a goal and see progress.

WEEK 04 Dashboard, Charts & Polish Days 22–30
📘 Learn First
  • Chart.js basics — bar chart, line chart via CDN
  • Pass C# data to JS (JSON.Serialize in Razor)
  • Bootstrap 5 grid layout for dashboard cards
  • Partial Views / ViewComponents in MVC
  • Streak calculation logic (consecutive days)
🔨 Then Build
  • Main dashboard: habit grid (like screenshot)
  • Streak counter per habit
  • Weekly habit completion bar chart
  • Calories & km progress bars with goals
  • Responsive layout — works on mobile
  • Clean up UI: consistent colors, icons
✅

Week 4 Goal: Full working app — habit grid, streaks, Google Fit stats, charts, goals. Ready to show or deploy.

// Beginner Tips — Read This
Learn → Build Loop Watch/read a concept for 30–60 min, then immediately code it. Don't watch 3 hours before coding.
Resources Use Microsoft Learn (free), Tim Corey on YouTube for ASP.NET, and official EF Core docs.
Git from Day 1 Commit every day. You will break things. Git lets you go back safely.
Google Fit API This is the hardest part. Give yourself extra time in Week 3. The Postman app helps test API calls first.
Don't over-design Get it working first. Bootstrap cards are fine. Polish in Week 4 only.
Stuck? That's normal Google the exact error message. Stack Overflow + GitHub Issues solve 90% of problems.
